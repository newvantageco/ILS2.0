## New API Endpoints
- GET /api/appointments/integrated - Enriched appointment list
- GET /api/appointments/integrated/:id - Single enriched appointment
- GET /api/appointments/queue/:stage - Queue by workflow stage
- POST /api/appointments/:id/check-in - Patient check-in
- POST /api/appointments/:id/start-exam - Start examination
- POST /api/appointments/:id/complete-exam - Complete exam & trigger dispense
- POST /api/appointments/:id/create-order - Link order to appointment
- PATCH /api/appointments/:id/status - Update status with WebSocket
